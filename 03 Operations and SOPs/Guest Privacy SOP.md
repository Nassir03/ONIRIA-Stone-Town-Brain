# Guest Privacy SOP

## Purpose

Protect guest identity, preferences, contact details, payments, health information, room status, and private requests.

## Core Rules

- Access only the guest data required for the Action.
- Do not share guest data in public channels.
- Do not expose room numbers unnecessarily.
- Treat allergies, spa notes, preferences, complaints, and payment information as sensitive.
- Record consent and REFERENCE where required.
- Escalate suspected privacy breaches.

## Procedure

1. Identify the purpose for accessing guest data.
2. Check whether the user or staff member is authorised.
3. Retrieve the minimum necessary information.
4. Use the approved communication channel.
5. Record sensitive changes.
6. Escalate uncertainty.
